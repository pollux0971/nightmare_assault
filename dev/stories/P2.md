# P2 · PromptComposer（決定性 prompt 組裝）

- **階段**：P（配置中心 / Story Agent 模組化 · patch v1.1）　**工線**：C（LLM Infra / Parser）
- **依賴**：P1　**契約**：PromptComposer, PromptFragments

## 目標 / 範圍
把 fragments 組成單一 compiled prompt：依 `sort_order` 排序、只取 enabled、變數代入、產**穩定 prompt_hash**、
**preview 不呼 LLM**、缺必填變數明確報錯。先不改 story agent 行為（P3 才接）。

## 對應來源
patch `docs/01 §P2`、`docs/08-integration-contracts.md`（prompt composition contract）、`task_cards/P2_prompt_composer.md`、`reference/pseudocode_reference.md`（範例，不照抄）。

## 實作步驟
- `core/config/prompt_composer.py`：compose(agent_name, profile_name, runtime_variables) → {compiled_prompt, prompt_hash, enabled_fragments, model_settings, context_policy}。
- 排序鍵 `sort_order`；只取 `enabled` binding；變數代入用安全模板（缺選填 → 可見 placeholder/warning，缺必填 → raise/驗證失敗）。
- prompt_hash 對「排序後 enabled fragment 內容 + 已代入變數」做穩定雜湊（同輸入同 hash）。
- preview 路徑全程零 LLM（注入 mock client 斷言未被呼叫）。

- [ ] 缺必填 runtime 變數 → preview/compile 失敗（不靜默跑成空白）；缺選填 → 可見 placeholder 或 warning。
