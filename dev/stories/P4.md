# P4 · Runtime 整合（config-first + static fallback）

- **階段**：P（配置中心 / Story Agent 模組化 · patch v1.1）　**工線**：D（Agent Logic）
- **依賴**：P3, SK04　**契約**：ConfigPromptSource, FeatureFlags, BeatLoop

## 目標 / 範圍
把 story prompt 來源切成 **config-first**（active profile → default profile → static built-in fallback），
受 `ENABLE_CONFIG_CENTER` 控制；**config 載入失敗一律 log + 退 static fallback，不崩 MVP-A**。static prompt 不刪。

## 對應來源
patch `docs/01 §P4`、`docs/02 §P4 guardrail`、`docs/08`（config-first prompt source / failure fallback contract）、`task_cards/P4_runtime_integration.md`。
既有：SK04 beat loop 整合（flag/分流/fallback/log）、dev/CONTRACTS.md `BeatLoop`。

## 實作步驟
- story agent 取 prompt 改走 PromptComposer（active→default→static），flag `ENABLE_CONFIG_CENTER`（預設 OFF，本工單可開）。
- flag 優先序：`.env > active profile DB > default DB > hardcoded`。
- 每場/每 beat log active profile + compiled prompt_hash；config 失敗時記 recoverable error、退 static、標 run warning。
