# Player Sovereignty 原則（系統共同前提）

> 這份是**所有 agent / gate 的共同前提**。Nightmare Assault 是**開放式探索遊戲**，
> 不是分支劇情遊戲。Narrative Control 的職責是「讓世界回應玩家」，**不是**「把劇情推向主線」。
>
> 一句話：**你要強制的是「世界要回應玩家」，不是「劇情要走向主線」。**

---

## 核心原則（硬規則）

1. **玩家可以拒絕主線。** 可以探索、撤退、繞路、破壞、躲藏、誤判、放棄調查。
2. **系統不得把玩家硬拉回唯一主線。** 沒有「正確路線」。
3. **系統只保證「行動有後果」，不保證「玩家接近真相」。** 每個行動都要在世界裡留下狀態後果（真相 / 假線索 / 地圖 / 道具 / 危險 / NPC 關係 / 時間…），但不強制其中要有真相。
4. **真相揭露是玩家探索的結果，不是系統強制派發的內容。**
5. **結局只能由「玩家明確確認」或「不可逆後果」觸發**，不得由敘事節奏 / 吸引子拉力自動觸發。
   - 允許的結局來源：玩家明確選擇「結束本次調查」、warden 硬觸發（致命/不可逆）。
   - **不允許**：危險值累積自動死亡、真相累積自動收束、逃脫拉力自動結局。
6. **玩家說「離開」時，預設不直接 ending。** 先分辨他是「離開房間 / 暫時撤退 / 結束本次調查」——不確定就**問**（見 ExitResolver）。
7. **`0/X` 真相不是失敗，而是低資訊結局（low-information ending）。** 不渲染成 fail screen / 考卷分數；語氣是「你活著離開，但沒有理解真相」，不是「你輸了」。

---

## 各模組的職責邊界（依此原則）

| 模組 | 該做 | 不該做 |
|---|---|---|
| **ProgressKernel** | 讓世界對玩家行動產生 affordance / 後果 | 選「下一個主線事件」、強制狀態收束 |
| **RevealBridge** | 有調查行為 → 產生 evidence（可為 truth / location / item / npc / hazard / exit clue，或 unmapped） | 強制每個行動都推進 truth；把無調查的 0/X 當錯誤 |
| **EndingGate / ExitResolver** | 玩家明確要結束時才結算；不確定時**呈現選擇** | 玩家一說離開就 ending；用拉力自動收束 |
| **StoryAgent** | 給**不同取向**的合理行動（推進調查 / 保命撤退 / 探索旁路 / 使用物品 / 自由輸入） | 每次都給「去主線 / 問 NPC / 查線索」引導選項 |
| **NPC-chat** | 給**可行動資訊**（B 區怎麼走、別走北門…），不一定是 truth | 一直用氣氛打斷、強迫吐主線真相 |

---

## ExitResolver 行為（不自動猜，偏向問）

玩家表達離開意圖時，**不要自動分類成 ending**：

- **明確「結束本次調查 / 接受結果 / 放棄目標」** → `run_ending` → EndingGate。
- **只是離開房間 / 區域** → `area_transition` → 換場景，遊戲繼續。
- **暫時撤退** → `temporary_retreat` → 降即時危險，遊戲繼續。
- **語意不明確（如只說「我離開」「找出口」）** → 輸出 **ExitOffer**（四選一，預設分支）：
  1. 離開目前區域，繼續探索
  2. 暫時撤退到安全處整理線索
  3. 結束本次調查，接受目前結果
  4. 回頭繼續尋找（你來的目的）

**任何時刻都一定有「明確結束本次調查」這個選項** —— 玩家永遠不會被困在「想走走不掉」。

## 真相進度的解讀（0/X 合法）

```text
0/X：玩家逃了，但一無所知        ← 合法（低資訊結局）
2/X：玩家知道一點，但可能誤判    ← 合法
4/X：玩家帶著部分真相離開        ← 合法
X/X：玩家理解全貌，能做高代價選擇 ← 合法
```

錯的不是 `0/X`，錯的是**「玩家明明調查了，卻 0/X」**（調查無回饋）。
驗收改成：**有調查行為 → 應產生 evidence 或 unmapped evidence；無調查行為 → 可以不推 truth。**

> 你的系統應該尊重玩家自由，但不能讓玩家的自由掉進空洞。
> 要修的不是「讓故事收束」，而是「讓每個選擇都有世界回應」。
